# Output Standards

Use these standards when packaging logo concepts or final logo assets.

## Exploration Output

For each style direction, include:
- Style name
- One-sentence concept
- Visual prompt or SVG construction note
- Main palette
- Where it works best
- Main risk

When using image generation, produce one focused concept board per direction. The board should show the primary mark, wordmark, one icon preview, and one light or dark usage preview.

## Final Asset Recommendations

For a selected direction, prepare or recommend:
- Primary horizontal logo
- Stacked logo
- Icon-only mark
- Monochrome black version
- Reversed white version
- Transparent PNG preview
- SVG source or SVG-ready construction notes
- README or presentation preview images for GitHub

## GitHub Showcase Assets

Include images that help a viewer understand the skill quickly:
- One gallery image per style direction
- One combined comparison image if available
- A short README section explaining what each style is for

Keep showcase files under `assets/showcase/` inside the skill folder so README links remain portable.

## Validation Checklist

Before calling a logo direction strong, check:
- It reads at small size
- It has a clear silhouette
- It works in monochrome or can be reduced to monochrome
- It does not rely only on glow, texture, or 3D material
- It avoids generic AI symbols unless intentionally requested
- It does not copy the logo of a real company
- Text is spelled correctly in preview boards
